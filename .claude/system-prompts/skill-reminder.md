# SessionStart Hook强化 - Skill使用提醒

> **在会话开始时强制提醒Skill使用规则**
> **创建时间**: 2025-01-11

---

## 🎯 目的

确保AI在每次会话开始时都明确知道：
1. 有哪些Skill可用
2. 何时必须使用Skill
3. 不使用Skill的后果

---

## 📋 当前可用的Skill

| Skill | 用途 | 强制使用场景 |
|-------|------|-------------|
| **docs-write** | 文档撰写 | 编写/更新.md文档 |
| **docs-review** | 文档审核 | 审核文档质量 |
| **hook-development** | Hook开发 | 创建Hook配置 |
| **command-development** | 命令开发 | 创建命令工具 |

---

## 🚨 关键规则

### 规则1：文档任务必须用Skill

**文档任务包括**：
- ✏️ 编写新文档
- ✏️ 更新现有文档
- ✏️ 编辑.md文件
- 🔍 审核文档质量
- 🔍 审查PR

**必须**：
- ✅ 调用 docs-write 或 docs-review
- ❌ 不能直接用Write/Edit工具

### 规则2：检测机制

**每次Write/Edit工具前，问自己**：

```
这是文档任务吗？
  ↓
是 → 必须先调用docs skill
  ↓
否 → 可以直接使用工具
```

### 规则3：错误纠正

**如果发现没用Skill**：
1. ❌ 立即停止
2. 📝 记录错误
3. 🔄 重新使用Skill完成任务

---

## 💡 快速检查清单


### 每次Write/Edit工具前

```
✓ 我调用了docs skill吗？
  ✓ 是 → 继续
  ✓ 否 → 必须先调用skill
```

### 每次任务完成后

```
✓ 我用了正确的Skill吗？
  ✓ 是 → 完成
  ✓ 否 → 记录错误，下次改进
```
