# 开发实施流程

> **开发过程中的协作方式和进度追踪**

---

## 🎯 流程概览

```
确认方案 → 制定计划 → 执行开发 → 定期汇报 → 完成交付
```

---

## 📋 步骤1：制定开发计划

### 我会这样做

1. **使用TodoWrite创建任务清单**
   ```
   ✅ 明确任务目标
   ✅ 分解任务步骤
   ✅ 标记优先级
   ```

2. **向你展示计划**
   ```
   【开发计划】
   📋 任务：实现用户游戏提交功能
   ⏱️ 预计：2-3小时
   🎯 里程碑：
     1. 设计提交表单 (30分钟)
     2. 实现提交逻辑 (1小时)
     3. 添加验证和提示 (30分钟)
     4. 测试验收 (30分钟)
   ```

3. **等待你的确认**

---

## 📋 步骤2：执行开发

### 开发过程中

**我会**：
- ✅ 专注执行任务
- ✅ 每完成一个重要步骤告诉你
- ✅ 遇到决策点立即询问
- ✅ 定期汇报整体进度

**示例汇报**:
```
【进度更新】
✅ 已完成：提交表单设计
🔄 进行中：提交逻辑实现 (50%)
⏳ 待开始：验证和提示

【下一步】
继续实现提交逻辑，预计1小时内完成。
```

### 你可以

**随时**：
- 📋 查看当前任务清单
- 📊 要求查看进度
- 🔄 提出修改意见
- ⏸️ 叫停某个功能
- ➕ 添加新需求

**示例**:
```
你: 进度怎么样了？
我: [显示当前进度]

你: 我觉得XX地方改一下
我: [评估影响] → 实施修改
```

---

## 📋 步骤3：遇到问题

### 技术选型问题

**我不会问**：
- ❌ "用React还是Vue？"
- ❌ "用MySQL还是MongoDB？"

**我会给方案**：
- ✅ "我建议用X方案，原因是..."
- ✅ "对你来说的好处是..."
- ✅ "你觉得可以吗？"

**只在必要时才让你选择**，其他的我来做。

### 需求理解问题

**我会**：
- ❓ 追问细节
- ❓ 提供选项
- ❓ 或给出建议方案

**示例**:
```
我: 对于评分范围，你希望：
   A. 1-5分（简单）
   B. 1-10分（更细致）

   根据我们的情况，我建议1-10分，
   因为能更好地区分游戏质量。

你: 就用你的建议
```

### 技术难题

**我会**：
- 🔍 查找解决方案
- 💡 提供多个方案
- 📊 说明优缺点
- 🎯 推荐最佳方案

**示例**:
```
【技术问题】
用户头像上传需要处理图片压缩。

【方案对比】
方案A：前端压缩
  优点：快速，不占服务器资源
  缺点：可能影响页面性能

方案B：后端压缩
  优点：不影响用户页面
  缺点：服务器压力稍大

【我的建议】
方案A（前端压缩），因为用户上传频率不高，
对服务器压力影响小。

你觉得可以吗？
```

---

## 📋 步骤4：定期汇报

### 汇报频率

**我会**：
- ✅ 每完成一个重要功能告诉你
- ✅ 每天结束时总结今日进度
- ✅ 遇到需要决策的问题立即问你

**你可以**：
- ✅ 要求我更频繁/更少地汇报
- ✅ 主动询问进度

### 汇报格式

```
【今日进度】
✅ 已完成：用户注册功能、游戏提交功能
🔄 进行中：评分系统（60%）
⏳ 待开始：排名计算

【遇到问题】
❓ 无 / 或列出具体问题

【下一步】
继续开发评分系统，预计今天下午完成。

【预计完成时间】
今天18:00前完成所有功能
```

---

## 📋 步骤5：完成交付

### 开发完成后

**我会**：
1. ✅ 告诉你功能已完成
2. ✅ 说明"改了什么效果"（用你能理解的方式）
3. ✅ 提供测试指引
4. ✅ 等待你验收

**示例**:
```
【功能完成】
✅ 用户游戏提交功能已完成

【效果说明】
现在用户可以：
1. 点击"提交游戏"按钮
2. 填写游戏名称、描述
3. 上传游戏文件
4. 点击"提交"后，会看到"提交成功"提示
5. 自动跳转到游戏列表，自己的游戏会高亮显示

【测试指引】
你可以试试：
- 提交一个游戏，看看流程是否顺畅
- 检查自己的游戏是否高亮显示
- 看看有没有什么不满意的地方

【等待验收】
请测试一下，有任何问题告诉我。
```

## 💡 协作原则

### 对产品经理（蜡烛先生）

**关键原则**：
- 📌 **你看不懂代码没关系**，我会说明"改了什么效果"
- 📌 **你随时可以调整方向**，有问题随时说
- 📌 **不必担心打扰我**，有任何想法随时沟通

**你可以**：
- ✅ 主动询问进度
- ✅ 提出修改意见
- ✅ 添加新需求
- ✅ 叫停某个功能

### 对技术开发（Claude）

**我的承诺**：
- ✅ 不擅自做主（重要决策先问你）
- ✅ 不过度设计（代码简洁够用就好）
- ✅ 及时汇报（让你知道进度）
- ✅ 诚实透明（不确定就是不确定）
