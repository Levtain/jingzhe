## 📊 RICE评估框架

### RICE定义

- **R (Reach)**: 影响范围（多少人/功能受影响）
- **I (Impact)**: 影响程度（0.25-3分：轻微→巨大）
- **C (Confidence)**: 信心指数（%：数据支持程度）
- **E (Effort)**: 工作量（人月：所需时间）

### 计算公式

```
RICE分数 = (R × I × C) / E
```

## 🔢 评分标准

### Reach (影响范围)

| 分数 | 范围 | 说明 |
|------|------|------|
| 100 | 全部用户 | 所有参赛者+评审团+玩家 |
| 50 | 大部分 | 主要用户群（参赛者） |
| 25 | 部分 | 特定用户群（评审团/团队队长） |
| 10 | 少数 | 边缘场景 |
| 1 | 极少 | 特殊情况 |

### Impact (影响程度)

| 分数 | 程度 | 说明 |
|------|------|------|
| 3 | 巨大 | 核心功能，影响平台成败 |
| 2 | 高 | 重要功能，显著提升体验 |
| 1 | 中 | 一般功能，有改进作用 |
| 0.5 | 低 | 可选功能，锦上添花 |
| 0.25 | 轻微 | 微小改进，几乎无感知 |

### Confidence (信心指数)

| 百分比 | 程度 | 说明 |
|--------|------|------|
| 100% | 非常确定 | 有明确数据或用户反馈支持 |
| 80% | 确定 | 有较强逻辑支撑 |
| 50% | 中等 | 有一定依据，但存在不确定性 |
| 20% | 低 | 依据不足，需要验证 |
| <20% | 很低 | 纯猜测，需要调研 |

### Effort (工作量)

| 单位 | 说明 | 换算 |
|------|------|------|
| 1人月 | 1人工作1个月 | 4人周 |
| 3人周 | 3人工作1周 | 0.75人月 |
| 1人周 | 1人工作1周 | 0.25人月 |
| 1人天 | 1人工作1天 | 0.05人月 |

## 📝 评估示例

### 示例1: 成就系统

```
问题: 成就系统设计
Reach: 100 (影响所有参赛者)
Impact: 1 (中等，激励机制)
Confidence: 80% (有其他平台经验)
Effort: 0.5人月 (2人周)

RICE = (100 × 1 × 0.8) / 0.5 = 160
```

### 示例2: 反作弊系统

```
问题: 善意度与反作弊阈值
Reach: 100 (影响平台健康)
Impact: 3 (巨大，防止刷分)
Confidence: 50% (不确定性高)
Effort: 1人月 (需要调试)

RICE = (100 × 3 × 0.5) / 1 = 150
```

### 示例3: 前端动画效果

```
问题: 页面过渡动画
Reach: 50 (所有用户，但不影响功能)
Impact: 0.5 (低，视觉优化)
Confidence: 80% (明确需要)
Effort: 0.25人月 (1人周)

RICE = (50 × 0.5 × 0.8) / 0.25 = 80
```

## 🎯 优先级分级

根据RICE分数，将问题分为4个优先级：

### 🔴 P0 - 立即讨论 (RICE > 120)
- 必须在第一届开赛前确认
- 影响核心功能
- 高风险

**预计数量**: 8-12个问题
**预计时间**: 1-2周

### 🟡 P1 - 近期讨论 (RICE 60-120)
- 重要但不紧急
- 显著提升体验
- 中风险

**预计数量**: 15-20个问题
**预计时间**: 2-3周

### 🟢 P2 - 中期讨论 (RICE 30-60)
- 可选功能
- 优化类需求
- 低风险

**预计数量**: 12-15个问题
**预计时间**: 后续迭代

### ⚪ P3 - 后续讨论 (RICE < 30)
- 锦上添花
- 边缘场景
- 可延后

**预计数量**: 8-10个问题
**预计时间**: V2版本或根据反馈

## 🚀 使用流程

### 步骤1: 启动RICE评估

```bash
用户: 帮我评估这53个问题的优先级

我: 调用 product-manager-toolkit skill
   使用 RICE 框架逐个评估

输出: rice-assessment-results.md
- 每个问题的RICE分数
- 优先级排序
- 建议讨论顺序
```

### 步骤2: 生成评估报告

```bash
用户: 生成评估报告

我: 使用 rice_prioritizer.py 脚本
   自动计算分数和排序

输出:
- P0问题清单 (立即讨论)
- P1问题清单 (近期讨论)
- P2问题清单 (中期讨论)
- P3问题清单 (后续讨论)
```

### 步骤3: 开始高优先级问题讨论

```bash
用户: 开始讨论P0问题

我: 调用 product-requirements skill
   为P0问题生成PRD

输出: prd/{feature-name}.md
- 结构化需求文档
- 质量评分 > 90分

然后使用 /discuss 命令
确认具体设计决策
```
