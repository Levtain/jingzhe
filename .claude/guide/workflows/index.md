# 工作流程 - 快速索引

> **完整的协作流程指南**

---

## 📋 流程概览

```
需求确认 → 开发实施 → 验收反馈 → 迭代完善
```

---

## 🎯 快速查找

### 需要确认需求？

→ 查看 [需求确认流程](requirement.md)

**关键步骤**:
1. 你描述需求（自然语言）
2. 我确认理解
3. 我给出技术方案
4. 你确认方案
5. 进入开发

**示例格式**:
```
【需求类型】新功能
【场景描述】在什么情况下...
【期望效果】我希望...
【优先级】高/中/低
```

---

### 需要开发实施？

→ 查看 [开发实施流程](development.md)

**我的工作**:
- 使用TodoWrite追踪任务
- 每完成一个模块向你展示
- 遇到决策点立即询问

**你可以**:
- 📋 查看当前任务清单
- 📊 要求查看进度
- 🔄 提出修改意见
- ⏸️ 叫停某个功能
- ➕ 添加新需求

---

### 需要验收反馈？

→ 查看 [验收反馈流程](review.md)

**发现问题格式**:
```
【问题类型】功能错误 / 体验不好 / 需求理解有误
【场景描述】我点击XX按钮后...
【期望效果】我希望...
【实际效果】现在的情况是...
```

**我的处理**:
1. 复述确认问题
2. 定位问题原因
3. 提供解决方案
4. 实施修改
5. 请你再次验收

---

### 需要了解沟通机制？

→ 查看 [沟通机制](communication.md)

**汇报频率**:
- ✅ 每完成一个重要功能告诉你
- ✅ 每天结束时总结今日进度
- ✅ 遇到需要决策的问题立即问你

**决策机制**:
- ❌ 不会让你选技术细节（"用React还是Vue？"）
- ✅ 会给你完整方案（"我建议用X，原因是...你觉得可以吗？"）

**沟通风格**:
- ✅ 用中文沟通
- ✅ 用产品经理能理解的方式解释技术
- ✅ 诚实透明（不确定就说不确定）

## 🚀 常见场景速查

| 场景 | 你怎么做 | 我怎么做 |
|-----|---------|---------|
| 提新需求 | 描述你想要什么 | 确认理解 → 给方案 → 等你确认 |
| 问进度 | "现在做到哪了？" | 告诉你当前状态和已完成事项 |
| 发现问题 | 描述问题场景 | 确认问题 → 找原因 → 给解决方案 |
| 想要修改 | "我觉得XX需要改一下" | 确认修改点 → 评估影响 → 实施修改 |
| 不确定需求 | "我不确定该怎么弄" | 提供选项让你选，或听我的建议 |
| 看不懂代码 | 不需要看代码 | 我用"人话"解释效果 |

## 📊 流程图

```
┌─────────────────────────────────────┐
│   你：用自然语言表达需求              │
│   我：确认理解，用产品经理语言反馈    │
│   你：确认方案                       │
│   我：执行开发，定期汇报进度          │
│   你：验收成果，提出修改意见          │
│   我：迭代完善                       │
└─────────────────────────────────────┘
```

**核心理念**:
- 你负责"做什么"（产品功能）
- 我负责"怎么做"（技术实现）
- 中间充分沟通确认
