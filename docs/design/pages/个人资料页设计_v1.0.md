## 设计概述

### 页面目标

个人资料页是用户展示个人信息、查看个人数据、管理个人作品的核心页面。该页面需要:

1. ✅ 展示用户基本信息（头像、昵称、简介等）
2. ✅ 展示用户身份信息（参赛者/评审团/硬核玩家等）
3. ✅ 展示个人作品列表（已提交的游戏）
4. ✅ 展示自我认知分析（心理价位吻合度统计）⭐
5. ✅ 展示金币账户信息（余额、冻结金币、交易记录）
6. ✅ 提供个人设置入口

### 用户角色与权限

| 查看者 | 本人 | 其他用户 |
|-------|------|---------|
| 查看基本信息 | ✅ | ✅ |
| 查看作品列表 | ✅ | ✅ |
| 查看评分统计 | ✅ | ✅ |
| 查看收益统计 | ✅ (详细) | ✅ (简化) |
| 查看金币余额 | ✅ | ❌ |
| 编辑个人资料 | ✅ | ❌ |
| 管理团队 | ✅ (队长) | ❌ |

#### 1.2 用户信息展示

**基本信息**:
- **头像**: 200x200px，点击可更换（本人）
- **昵称**: 用户设置的自定义昵称
- **身份标签**:
  - 参赛者: 蓝色标签
  - 评审团: 紫色标签
  - 硬核玩家: 绿色标签
  - 团队队长: 橙色标签
- **个人简介**: 用户填写的自我介绍（最多200字）

**统计数据**:
- **浏览量**: 个人资料页访问次数
- **作品数**: 已提交的游戏数量
- **评分数**: 对其他游戏的评分次数

#### 1.3 身份标签规则

| 身份 | 显示条件 | 样式 |

| 评审团 | 当前是评审团成员 | 紫色 |
| 硬核玩家 | 已取得硬核玩家身份 | 绿色 |
| 团队队长 | 是某个团队的队长 | 橙色 |
| 作弊者 | 有作弊记录 | 红色 |

**作弊者标签** (特殊处理):
- 样式: 红色"作弊者"徽章
- 永久显示，无法隐藏
- 点击可查看作弊详情

#### 2.2 核心指标设计 ⭐⭐⭐

**所有作品平均吻合度**:
- 计算公式: 所有作品的吻合度算术平均
- 显示格式: 百分比 + 状态标识
- 状态标识:
  - ≥90%: "精准预测 🎯"
  - 70-89%: "接近预期 ✅"
  - 50-69%: "略有偏差 ⚠️"
  - <50%: "差距较大 ❌"

**趋势提示**:
- 吻合度上升: "你的自我认知越来越准确！ 🎉"
- 吻合度下降: "继续加油，多参考市场数据 💪"
- 首次参赛: "欢迎参赛！积累经验提升定价能力 🌱"

#### 2.3 历史作品记录

**每条记录包含**:
1. **游戏标题和赛季**
2. **心理价位 vs 实际收益**
3. **吻合度百分比**
4. **溢价/折价标签**
5. **打赏统计** (人数、总额)

**点击交互**:
- 点击游戏标题 → 跳转到游戏详情页
- 点击吻合度 → 显示计算公式弹窗

#### 2.4 定价能力成长曲线

**数据可视化**: 线性图表

**X轴**: 赛季编号（第1赛季、第2赛季、...）
**Y轴**: 平均吻合度（0-100%）

**图表特性**:
- 显示所有赛季的数据点
- 用折线连接各数据点
- 显示成长趋势（上升/下降/平稳）

**示例**:

## 响应式设计

### 桌面端 (≥1024px)

- 左右布局: 左侧主区域70%，右侧边栏30%
- 自我认知分析面板全宽显示
- 作品列表横向排列卡片

### 平板端 (768px - 1023px)

- 单列布局: 主区域和边栏垂直排列
- 自我认知分析面板保持全宽
- 边栏卡片移到主内容下方

### 移动端 (<768px)

- 单列布局
- 统计数据卡片改为单行滑动
- 作品卡片简化（隐藏部分信息）
- 边栏卡片简化或折叠

## 性能优化

### 首屏加载

**必需数据**:
1. 用户基本信息
2. 身份标签
3. 作品列表（最近3个）
4. 金币余额（本人）

**加载顺序**:
1. 骨架屏占位
2. 加载用户基本信息
3. 加载作品列表（异步）
4. 加载自我认知分析（异步，仅本人）
5. 加载金币账户（异步，仅本人）

### 延迟加载

**非首屏数据**:
1. 评分历史（用户滚动到底部时加载）
2. 交易记录（用户点击"查看全部"时加载）
3. 成长曲线图（用户滚动到可见区域时加载）

### 缓存策略

**缓存内容**:
- 用户基本信息: 缓存5分钟
- 作品列表: 缓存5分钟
- 金币余额: 缓存1分钟（本人）

**缓存失效**:
- 用户编辑资料后 → 清除基本信息缓存
- 用户提交新作品后 → 清除作品列表缓存
- 用户交易后 → 清除金币余额缓存
