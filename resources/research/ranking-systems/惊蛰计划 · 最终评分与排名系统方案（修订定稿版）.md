# 惊蛰计划 · 最终评分与排名系统方案（修订定稿版）

## 一、系统设计总体原则

本方案服务于商业化可运营的线上赛事平台，其核心目标是：

- 公平、公正、可解释

- 对抗恶意评分与组织化刷分

- 避免优质作品因冷启动被埋没

- 保证第一届也能稳定运行

- 支持后续扩展与升级

系统采用**双轨排名架构**：

- 评委榜（Judge Rank）：专业评价导向

- 玩家榜（Player Rank）：大众口碑导向

两套系统独立运行与展示，互不稀释权重。

---

## 二、统一评分维度设计

示例评价维度（可按平台自定义）：

- 创意

- 玩法

- 美术

- 音效

- 叙事

- 完成度

每个维度：

- 单独统计与排名

- 参与综合评分计算

总榜为**加权综合得分**，所用权重公开透明。

## 三、玩家榜（Player Rank）最终算法

玩家榜用于反映大众口碑，目标：

- 对抗组织化刷分

- 避免恶意打低

- 冷启动保护新作品

- 鼓励玩家多评价但避免滥评刷权重

最终得分计算流程：

`原始评分 → 用户贡献权重 → 双侧截尾消除极端值 → 鲁棒均值（Huber Mean） → 贝叶斯平滑（冷启动保护） → 争议度惩罚（标准差调整） = 玩家最终得分 P-Score`

### 1. 用户贡献权重设计

目的：

- 鼓励认真评价

- 抑制批量注册

定义权重：

`Wu = 1 + ln(1 + nu)`

其中：

- nu = 该玩家有效评分次数

- ln = 自然对数

性质：

- 多评多权

- 权重递减（防止滥评膨胀）

- 不依赖历届数据（适配第一届）

### 2. 加权评分

每个评分被转换为：

`rij' = rij × Wu`

### 3. 双侧截尾（防极端打分与亲友团）

对每个作品：

- 排序评分

- 去掉最高 α%

- 去掉最低 α%

推荐参数：

`α = 10% 评分数 < 10 不启用截尾`

有效抑制：

- 纯 1 分轰炸

- 纯 10 分亲友团

### 4. 鲁棒均值（Huber）

在截尾后的集合中计算鲁棒均值 RH：

优势：

- 极端值影响减弱

- 正常评分影响接近均值

避免：

- 中位数阶梯化问题

- 简单均值的脆弱性

### 5. 贝叶斯平滑：冷启动保护

用于避免“好游戏无人看 → 排名垫底”

公式：

`PlayerScore = ( n × RH + k × μ ) / ( n + k )`

其中：

- n = 实际评分数

- μ = 平台总体均值

- k = 平滑强度（推荐 10–20）

效果：

- 少量评分 → 靠近整体平均

- 大量评分 → 靠近真实表现

### 6. 争议度惩罚（识别分歧作品）

定义 σ：

- 标准差或鲁棒标准差

最终得分：

`P-Final = PlayerScore − λ × σ`

其中：

- λ = 惩罚权重（推荐 0.2–0.4）

意义：

- 两极分化 = 争议作品

- 高分高争议 → 自动下调

- 避免“极端粉丝 + 极端黑”冲顶

## 四、评委榜（Judge Rank）最终算法

评委榜强调：

- 专业性

- 可讨论性

- 小样本可靠性

不引入复杂平滑，不采用黑箱模型。

### 最终算法

1. 去掉最高分 1 个

2. 去掉最低分 1 个

3. 在剩余评分中取鲁棒均值（Huber）

特殊情况：

- 评委 ≤ 3 人，则不截尾

优势：

- 对单一评委的坏心或偏见免疫

- 保持分数可解释性

- 评委内部容易达成共识

标签标识：

- 玩家热爱

- 评委推荐

- 两极争议

- 冷启动潜力股

符合商业平台运营与传播需求。

## 六、抗刷分能力说明

| 可防御行为    | 机制          |
| -------- | ----------- |
| 亲友团10分刷榜 | 双截尾 + Huber |
| 恶意1分轰炸   | 双截尾 + Huber |
| 组织化水军    | 贡献权重限制      |
| 小样本冲榜    | 贝叶斯平滑       |
| 两级撕裂作品   | 标准差惩罚       |
| 早期评分失真   | 平滑回归均值      |
| 机器人批量小号  | 权重不足        |

系统无需：

- 历史信誉

- 行为特征数据

- 账号社交关系

适配首届赛事。

## 九、对外公开说明（简化版）

可对用户公开为简化版解释：

1. 评委榜与玩家榜分开算

2. 极端分会被剔除

3. 小样本不会直接冲榜

4. 分歧大的作品会下调

5. 多评价的玩家影响更大

内部算法可保留技术细节。

本最终修订版方案：

- 以 DRB-Rank 为主框架

- 融合：
