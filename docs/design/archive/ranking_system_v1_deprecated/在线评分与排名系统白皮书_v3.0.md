
# 在线评分与排名系统白皮书（工程实现版）
## ——面向 Game Jam / UGC 评价平台的抗刷分与冷启动优化方案

版本：v3.0（工程实现导向）  
用途：可直接用于实际平台产品化实现与团队评审  
作者：系统设计助手（基于用户交互定制）

# 0. 文档定位与阅读指南

本白皮书为**工程可落地版本**，目标不是追求理论上最强的统计严格性，而是在以下约束下达到最优折中：

- 必须可在实际平台中部署
- 必须在大规模用户场景下能实时计算
- 必须具有可解释性与用户可沟通性
- 必须具备抗刷分能力
- 必须显著缓解“好作品被埋没”问题
- 必须支持渐进增强（逐步升级算法体系）

本方案吸收了：

- 原始设计思想
- 技术评审报告提出的批评
- 复审意见 v2.0 的折中建议
- 平台级系统实现经验

最终形成**“鲁棒统计 + 动态贝叶斯平滑 + 信誉权重 + 分发机制”**的完整系统。

# 1. 系统总体设计目标

## 1.1 业务目标

本系统服务于以下类型的平台：

- Game Jam
- 作品征集评审
- Indie Game Contest
- Mod / 关卡创作竞赛
- 其他 UGC 评分活动

核心业务目标：

1. 支持大规模作品的评分与排名
2. 减少恶意刷分对排名的干扰
3. 识别并提升被低估的优质作品
4. 提供透明与可沟通的规则
5. 提供平台方可调节运营策略的参数接口

---

## 1.2 算法设计目标

算法上要达成：

- 抗噪声（鲁棒）
- 抗操纵（反作弊）
- 冷启动公平
- 不确定性识别
- 评分者信誉自适应调整
- 多维度评分支持

# 4. 反作弊与信誉系统设计

## 4.1 评分者信誉分（Rater Trust Score, RTS）

为每位评分者维护信誉评分 RTS ∈ [0,1]

影响因素：

| 指标 | 逻辑 | 效果 |
|------|------|------|
| 评分多样性 | 方差过低降权 | 防止清一色 5 星 | 
| 打分速度 | 过快可能未游玩 | 降权 |
| 共识偏离度 | 与群体偏离严重 | 降权 |
| 历史行为 | 投诉、撤销记录 | 降权 |
| 互评团发现 | 集群检测标记 | 强降权 |
| 评论有效性 | 字数、情感检测 | 升权 |

实现策略：

- 不“封禁”评分
- 仅影响其评分权重

---

## 4.2 互评团检测（Collusion Detection）

识别：

- 小团体互相高分
- 明显内外评分差异
- 相似投票模式

采用：

- 基于图的聚类
- 皮尔逊相关系数
- 谱聚类 + 显著性检验

系统行为：

- 自动降低该团体内部评分权重
- 自动提高外部真实评分权重
- 不提示用户（避免博弈）

# 5. 单一作品得分计算流程（核心）

本系统以以下四步计算每个作品最终得分：

1) 鲁棒均值计算（抗极端刷分）  
2) 动态贝叶斯平滑（解决评分不足与冷启动）  
3) 引入评分者信誉权重 RTS  
4) 计算不确定性并进行惩罚  

---

## 5.1 步骤一：鲁棒均值（Huber Mean）

采用 **Huber M-估计量**：

- 小残差：平方惩罚（类似均值）
- 大残差：线性惩罚（类似绝对值）

功能：

- 自动抑制极端恶意评分
- 无需手工设置截断比例
- 可解释性强

---

## 5.2 步骤二：评分者信誉加权

加权评分：

S = Σ(rating × RTS) / Σ(RTS)

效果：

- 信誉高 → 影响大
- 疑似水军 → 影响低

---

## 5.3 步骤三：动态贝叶斯平滑

避免：

- 新作少量高分 → 冲榜
- 老作被低估

平滑公式：

S' = (n × S + k × μ) / (n + k)

其中：

- S = 当前作品鲁棒均值
- n = 实际评分数
- μ = 系统全场平均
- k = 动态平滑系数（自动调节）

### 动态 k

k = k0 / confidence

confidence 来源：

- 评分一致性
- 方差
- 可信评分比例

---

## 5.4 步骤四：不确定性惩罚（μ − λσ 型）

最终得分：

Final = S' − λ × σ

其中：

- σ = 评分标准差（或后验方差）
- λ 为平台可调参数

含义：

- 高均值但不稳定 → 惩罚
- 高均值且稳定 → 奖励

# 6. 排名系统设计

提供三类排名：

1) 总榜（Final Score 排序）  
2) 分类榜（维度得分）  
3) 潜力榜（高均值但低曝光作品）  

支持：

- Pareto 排序
- 多目标排序
- 权重调参式排序

# 10. 示例计算（可直接演示）

假设：

- 全场均值 μ = 3.5
- 某作品评分：  
  5, 5, 5, 5, 1

普通平均：4.2 → 失真

### 1) Huber 鲁棒均值

输出 ≈ 4.6

### 2) 动态贝叶斯平滑 (k = 10)

S' = (5×4.6 + 10×3.5) / 15 = 3.9

### 3) 不确定性惩罚

σ ≈ 1.6

Final = 3.9 − 0.3×1.6 ≈ 3.42

解释：

- 高分但不稳定 → 惩罚
- 刷分无法冲榜
