## 📊 完整的Agent体系

### 设计阶段Agent (3个)

```yaml
1. discussion-agent (问题讨论)
   - 功能: 自动化问题讨论流程
   - 价值: 减少60%手动操作

2. design-audit-agent (设计审核)
   - 功能: 深度设计质量审核
   - 价值: 发现隐性风险和矛盾
   - 使用频率: 模块完成后1次
   - 核心特性: 5维审核,攻防推演,A/B/C分级

3. completion-check-agent (完成度验证)
   - 功能: 验证模块完整性
   - 价值: 确保质量标准
   - 使用频率: 里程碑时1次
   - 核心特性: 5维验证,TODO清单,明确标准
```

## 🚀 完整使用流程

### 日常使用流程

```yaml
每天开始:
  "开始今天的工作"
  → workflow-orchestrator-agent
  → 分析项目状态
  → 推荐今日任务
  → 自动启动相应Agent

设计讨论:
  "继续讨论"
  → discussion-agent
  → 自动加载问题
  → 自动记录决策
  → 询问是否继续

模块完成:
  "完成这个模块"
  → workflow-orchestrator-agent
  → 自动协调验证步骤
  → 确保质量标准

代码审核:
  "审核这段代码"
  → code-review-agent
  → 5维代码审核
  → 具体修复建议

代码生成:
  "生成代码"
  → code-generation-agent
  → 生成完整框架
  → 提供实现TODO

每天结束:
  "生成工作日报"
  → workflow-orchestrator-agent
  → 总结今日完成
  → 规划明日任务
```

### Agent协作关系

```yaml
中心协调者: workflow-orchestrator-agent
  ↓ 分析和推荐
  ↓ 使用Task工具调用
  ↓
设计阶段:
  ├─ discussion-agent (讨论问题)
  ├─ design-audit-agent (审核设计)
  └─ completion-check-agent (验证完整性)

开发阶段:
  ├─ code-generation-agent (生成代码)
  ├─ code-review-agent (审核代码)
  └─ completion-check-agent (验证完整性)

文档管理:
  └─ doc-sync-agent (同步文档)

反馈:
  所有Agent → workflow-orchestrator-agent
  → 更新状态 → 推荐下一步
```
