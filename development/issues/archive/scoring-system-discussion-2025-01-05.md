## 一、评分机制设计

### 1.1 评分颗粒度与统计
**当前**: 0-10整数档

**问题**:
- [x] 是否支持小数(如0.5分档)?如果不支持,如何保证6个维度×10分的总分有足够区分度?
- [x] 如果支持小数,是0.5还是0.1?这对后续排名精度有什么影响?
- [x] 评分后的显示方式:是显示原始分还是归一化到百分比/星级?

**已确认**:
- ✅ 支持0.5分档位
- ✅ 显示方式:星级 + 原始分(如 `★★★★☆ 8.5/10`)

---

### 1.2 评分资格与权限
**当前**: 参赛者与评审团可评分,路人不可

**问题**:
- [x] "参赛者"定义范围是什么?只要提交了游戏就可以评所有人的游戏?还是需要达到某个条件(如通过审核)?
- [x] 如果有参赛者被取消资格(如作弊),他之前给出的评分是否作废?
- [x] 评审团成员如果也提交了游戏,能否给自己评分?
- [x] ~~评审团成员之间能否互评?~~ (已删除:评审团不能提交游戏,不存在互评场景)

**已确认**:
- ✅ **Q1**: 只要在规定时间内提交了参赛游戏就可以评价他人的游戏
  - 参赛者及其团队成员都不能评价自己的游戏
  - 如果发现违规,该团队作品的评分将被处理,但他们对其他人的评分保留(难以追溯,不予处理)

- ✅ **Q2**: 参赛者被取消资格后,其评分不作废,保留但标注"已失效账号"
  - **渐进式惩罚策略**:
    - 首次作弊: 取消本次评选资格 + "作弊者"成就永久公示
    - 二次作弊: 注销账号 + 删除所有游戏 + 禁止邮箱/手机号/设备指纹再次注册
  - **支持申诉机制**: 每人仅限1次申诉,7个工作日内复核
  - **技术实现简单**: 不需要重新计算评分,作弊者之前的评分保留

- ✅ **Q3**: 评审团成员不能同时参赛(身份互斥)
  - **原因**: 既当选手又当裁判员不公平
  - **规则**: 选择成为评审团后,该账号无法提交参赛游戏
  - **技术实现**: 评审团账号在游戏提交功能处禁用或隐藏

- ✅ **Q4**: ~~评审团成员之间能否互评~~ 已删除,因评审团无法提交游戏

**补充确认**:
- ✅ 评审团成员只能评价参赛者的游戏,不能评价其他评审团(因为没有游戏)
- ✅ 评分权限已明确:参赛者+评审团可评分,玩家/路人/管理员不可评分

### 1.3 评分时机与修改
**当前**: 未明确

**问题**:
- [x] 游戏提交后是否立即开放评分?还是需要审核通过后?
- [x] 评分是否允许修改?如果允许,修改次数限制?时间限制?
- [x] ~~如果游戏更新了版本,之前的评分是否保留?是否需要重新评分?~~ (已删除:开发期结束后不可修改版本)
- [ ] 互玩互评期12天,最后几天如果突然有大量评分涌入(战术性评分),如何处理?

**已确认**:
- ✅ **Q1**: 游戏提交后立即开放评分,无需审核
  - **原因**: 目前阶段无需审核流程
  - **优势**: 简化流程,用户提交后即可获得反馈

- ✅ **Q2**: 允许修改评分,直到评分阶段结束前都可以修改
  - **无次数限制**: 用户可以根据游戏更新或重新思考后调整评分
  - **时间限制**: 评分阶段结束后锁定所有评分
  - **优势**: 灵活性高,鼓励用户深入体验后调整评价
  - **交互流程**:
    - 允许全部维度评0分
    - 必须点击"提交评分"按钮才生效
    - 提交前需二次确认(弹窗显示评分详情,确认是否提交)
    - 修改评分需点击"修改评分"按钮
    - 修改后也需二次确认才能生效

- ✅ **Q4**: 战术性评分/刷分防护机制
  - **时间缓冲**: 互评期结束后延后2天再公布结果,给申诉和复核留出时间
  - **监控机制**: 后台实时监控评分频率和评分分布
  - **高危监测触发条件**: 10分钟内评分超过6个游戏
  - **警告触发条件**: 高危监测用户 + 6个维度每个维度评分均<2分
  - **分级处理**:
    - 高频评分(无恶意倾向): 站内信警告
    - 高频差评(恶意战术性评分): 直接冻结账号,按作弊处理(Q2的渐进式惩罚)
  - **评审团特殊处理**: 更严格标准
    - 发现任何违规行为(后台监测或用户举报)
    - 经审核无误后直接取消评审团资格
    - 永久禁止再次成为评审团成员
    - 禁止参赛
  - **游玩时间**: 暂不统计(因游戏仅提供下载链接,无法追踪游玩时长)
  - **透明度**: 向被处罚用户说明原因和证据

### 1.4 自评规则
**当前**: 自评允许性可配置

**问题**:
- [x] 为什么考虑"可配置"?什么场景下允许,什么场景下不允许?
- [x] 如果允许自评,自评的权重是否与其他评分相同?
- [x] 如果不允许自评,UI如何处理?是隐藏自己的游戏还是显示但不可评?
- [x] 团队赛时,团队成员能否给团队游戏评分?

**已确认**:
- ✅ **Q1-Q4**: 禁止自评和自赏,但允许评论区互动
  - **禁止行为**: 参赛者及其团队成员不能给自己的游戏评分(0-10分)或打赏金币
  - **允许行为**: 参赛者可以在自己游戏的评论区写评论、回复其他人、进行互动交流
  - **原因**:
    - **公平性**: 避免自我炒作评分或恶意打赏刷金币
    - **数据质量**: 自评数据往往主观性过强,影响排名准确性
    - **社区互动**: 作者在评论区与玩家交流是正常且有益的
  - **UI处理**: 在游戏列表/详情页中,自己的游戏评分和打赏功能显示"不可操作",但评论区正常可用
  - **团队赛规则**: 团队所有成员都不能给团队游戏评分或打赏,但可以参与评论区互动

## 二、评分维度与特殊处理

### 2.1 AI使用时的维度缺失
**当前**: 使用AI→不参与视觉/音频维度评分

**问题**:
- [x] 如何判断"使用AI"?是作者自行申报?系统检测?还是用户举报?
- [ ] 如果作者申报使用AI制作美术,但用户认为没有使用AI(或反之),如何处理争议?
- [ ] 缺失的维度在总分计算时如何处理?
  - 选项A: 总分满分降低(如60分→50分)
  - 选项B: 未评分维度记0分
  - 选项C: 按剩余维度平均分补偿
  - 选项D: 完全不显示总分,只显示各维度得分
- [ ] 如果某游戏使用AI制作美术,但也有原创音乐,是否"音乐音频"维度正常评分但"视觉效果"不评?
- [ ] 如果游戏完全使用AI生成(无任何原创内容),是否允许参赛?如果允许,如何评分?
- [ ] AI使用的程度是否需要区分?(部分使用 vs 全部使用)

**已确认**:
- ✅ **Q1**: 要求作者自行申报 + 接受用户举报
  - 提交时必须勾选AI使用情况:
    - 美术: 全原创 / 部分使用AI / 全部使用AI
    - 音频: 全原创 / 部分使用AI / 全部使用AI
  - 如果选择"使用AI",必须说明使用了哪些AI工具
  - 接受用户举报,举报时需提供证据
  - 人工复核后,如果发现虚假申报,取消参赛资格

- ✅ **Q2**: AI申报争议处理机制
  - **情况A**: 作者申报使用AI,但用户认为没有使用
    - 不做讨论,不符合逻辑(作者已诚实申报)
  - **情况B**: 作者承诺未使用AI,但被用户举报
    - **分级处理**:
      - 举报次数<5次: 站内信通知作者自证清白
      - 举报次数≥5次: 全频道广播该事件,由评审团和管理员进行鉴别仲裁
    - **仲裁结果**:
      - 确认使用AI: 取消参赛资格 + "作弊者"标签
      - 确认原创: 恢复名誉,举报记录清除

- ✅ **Q3**: 革命性排名系统 - 单维度排名 + 优选游戏机制
  - **核心原则**: 不计算总分,仅针对每个维度进行单一排名
  - **榜单结构** (6个维度 × 2个评分群体 = 12个榜单):
    - 创新性 (参赛者榜) / 创新性 (评审团榜)
    - 主题诠释 (参赛者榜) / 主题诠释 (评审团榜)
    - 视觉效果 (参赛者榜) / 视觉效果 (评审团榜)
    - 音乐音频 (参赛者榜) / 音乐音频 (评审团榜)
    - 整体性 (参赛者榜) / 整体性 (评审团榜)
    - 有活儿 (参赛者榜) / 有活儿 (评审团榜)
  - **优选游戏判定标准**:
    - 参赛者优选游戏: 3个及以上维度进入参赛者榜前10
    - 评审团优选游戏: 3个及以上维度进入评审团榜前10
  - **展示方式**:
    - 各游戏在所属维度榜单中正常显示
    - 优选游戏额外获得"优选游戏"标签
    - 设置独立的"优选游戏"榜单(参赛者版 / 评审团版)
  - **优选游戏缺失处理**: 宁缺毋滥,若某届无游戏达标则不设置
  - **AI维度缺失完美解决**:
    - 使用AI的维度直接不参与该维度排名
    - 其他维度正常竞争,不影响优选游戏判定
    - 避免了总分计算的所有争议问题

- ✅ **Q4-Q6**: AI使用的其他规则
  - **Q4**: 部分维度使用AI的处理
    - 如美术使用AI但音乐原创: "音乐音频"维度正常评分排名,"视觉效果"不参与
  - **Q5**: 完全AI生成游戏(无任何原创内容)
    - 允许参赛
    - 视觉效果和音乐音频维度不参与排名
    - 其他维度(创新性、主题诠释、整体性、有活儿)正常参与
  - **Q6**: AI使用程度区分
    - 通过申报机制已解决: 全原创 / 部分使用AI / 全部使用AI
    - 不同情况对应不同的维度参与情况

### 2.2 维度定义的主观性
**当前**: 创新性、主题诠释、视觉效果、音乐音频、整体性、整活儿

**问题**:
- [x] 这些维度如何定义?"创新性"vs"整活儿"的边界是什么?
- [ ] 是否提供评分指南?如"创新性"的评价标准是什么?
- [ ] 如果参赛者对某维度不理解(如"整活儿"),是否会影响评分公平性?
- [ ] 6个维度是否可以选填?还是必须全部评分?

**已确认**:
- ✅ **Q1**: 6个评分维度的详细定义

  **1. 创新性 (Innovation)**
  > 评价游戏在玩法、机制、叙事或技术上的原创性和突破性。是否提供了新颖的体验?是否突破了传统游戏的框架?

  **核心关注点**:
  - 玩法机制的创新性
  - 叙事手法的新颖性
  - 技术实现的突破性
  - 对现有游戏类型的超越

  **评分参考**:
  - 10分: 革命性创新,开创全新玩法/机制
  - 8-9分: 高度创新,有显著的新颖性
  - 6-7分: 有一定创新,但仍在传统框架内
  - 4-5分: 创新不足,模仿痕迹明显
  - 0-3分: 缺乏创新,完全模仿或陈词滥调

  **边界说明**: 创新性不等于"整活儿",创新更强调技术和设计的突破,而"整活儿"更强调趣味性和娱乐性。

  ---

  **2. 主题诠释 (Theme Interpretation)**
  > 评价游戏对比赛主题的理解程度和表达质量。是否紧扣主题?是否有独特的视角?主题融入是否自然?

  **核心关注点**:
  - 对主题的理解深度
  - 主题与玩法的结合程度
  - 主题表达的创意性
  - 主题融入的自然度(不生硬)

  **评分参考**:
  - 10分: 主题诠释完美,独特视角且深度融合
  - 8-9分: 主题诠释优秀,理解深刻且表达自然
  - 6-7分: 主题诠释良好,基本符合要求
  - 4-5分: 主题诠释一般,牵强或表面化
  - 0-3分: 主题诠释失败,偏离主题或完全忽视

  **边界说明**: 主题诠释关注"是否切题",而创新性关注"是否新颖",两者可以独立评价。

  ---

  **3. 视觉效果 (Visual Art)**
  > 评价游戏的美术风格、画面表现力、视觉设计质量。包括UI设计、角色设计、场景设计、动画效果等。

  **核心关注点**:
  - 美术风格的统一性和美感
  - 画面表现的精致程度
  - 视觉设计对游戏体验的贡献
  - UI/UX设计的合理性

  **评分参考**:
  - 10分: 视觉效果卓越,风格独特且完美契合
  - 8-9分: 视觉效果优秀,画面精美且风格统一
  - 6-7分: 视觉效果良好,基本完成但不出彩
  - 4-5分: 视觉效果一般,有明显缺陷
  - 0-3分: 视觉效果差,严重影响体验

  **特殊情况**: 如果使用AI制作美术,此维度不参与排名。

  ---

  **4. 音乐音频 (Music & Audio)**
  > 评价游戏的音效设计、背景音乐、配音质量、音频与玩法的配合度。

  **核心关注点**:
  - BGM的质量和氛围营造
  - 音效的设计和反馈性
  - 音频对游戏体验的增强作用
  - 音频与玩法的协调性

  **评分参考**:
  - 10分: 音频完美,极大提升游戏沉浸感
  - 8-9分: 音频优秀,质量高且配合好
  - 6-7分: 音频良好,基本满足需求
  - 4-5分: 音频一般,有明显不足
  - 0-3分: 音频差,缺失或严重影响体验

  **特殊情况**: 如果使用AI制作音频,此维度不参与排名。

  ---

  **5. 整体性 (Completeness)**
  > 评价游戏的完成度、各元素的协调性、整体设计的成熟度。是否是一个完整、可玩、无明显Bug的游戏?

  **核心关注点**:
  - 游戏的完整性和可玩性
  - 各系统(玩法、美术、音频、UI)的协调性
  - Bug数量和严重程度
  - 游戏的打磨程度

  **评分参考**:
  - 10分: 完美完成,高度协调且无明显问题
  - 8-9分: 完成度高,协调性好且打磨精致
  - 6-7分: 完成度尚可,有小瑕疵但不影响体验
  - 4-5分: 完成度一般,有明显缺陷或Bug
  - 0-3分: 完成度低,严重Bug或未完成

  **边界说明**: 整体性关注"完成度和协调性",而"整活儿"关注"趣味性和娱乐性",一个游戏可以完成度高但不有趣,或者有趣但完成度一般。

  ---

  **6. 整活儿 (Fun Factor / Meme Value)** ⭐
  > 评价游戏是否"整活儿成功"——即使不完美,但通过巧思、幽默、创意等让游戏充满趣味性、娱乐性或令人捧腹。重视游戏的灵魂和惊喜感。

  **核心关注点**:
  - 游戏的趣味性和娱乐性
  - 是否有令人惊喜的巧思
  - 幽默感或玩梗(适度)的运用
  - 让人会心一笑或拍案叫绝的设计

  **评分参考**:
  - 10分: 整活儿大成功! 玩法/剧情充满惊喜,令人捧腹或惊叹,玩了还想玩
  - 8-9分: 整活儿成功,有很多巧思和有趣的设计,玩得开心
  - 6-7分: 有一些有趣的点,但整体不够突出
  - 4-5分: 整活儿失败,强行搞笑或创意尴尬,感觉尴尬
  - 0-3分: 没有整活儿的意图,或者整活儿翻车,完全不好玩

  **边界说明**:
  - "整活儿"vs"创新性": 创新强调技术/设计突破,整活儿强调趣味性/娱乐性
  - "整活儿"vs"整体性": 整体性关注完成度,整活儿关注好不好玩
  - 一个游戏可以不创新(传统玩法)、不完美(有Bug),但"整活儿成功"(超级好玩)

  **文化内涵**: "整活儿"是游戏开发社区的网络用语,指通过巧思、创意让作品充满趣味性,即使技术不完美也能打动人心。

  ---

  **维度关系总结**:
  - **创新性**: 是否新颖? (技术/设计突破)
  - **主题诠释**: 是否切题? (对主题的理解和表达)
  - **视觉效果**: 美术好不好看? (美术质量)
  - **音乐音频**: 音频好不好听? (音效质量)
  - **整体性**: 是否完整? (完成度和协调性)
  - **整活儿**: 是否好玩? (趣味性和娱乐性) ⭐

  **6个维度相互独立,各有侧重,共同构成对游戏的全面评价。**

### 4.2 评审团权重
**当前**: ×2

**问题**:
- [x] 为什么是×2?这个权重如何得出?
- [x] 是否意味着"1个评审团=2个参赛者"?如果是,为什么评审团意见更重要?
- [x] 评审团权重是否在不同维度上不同?如"创新性"维度评审团权重×3?
- [x] 如果某评审团未评分,是否按"×0"计算还是直接不计入?

**已确认**:
- ✅ **Q1-Q4**: 评审团权重规则
  - **权重**: ×2(固定)
  - **重要澄清**: 评审团权重×2 **只影响"评分人次"统计,不影响具体分值计算**
  - **含义**:
    - 评审团的每个评分在计算"评分人次"时按2人次计算
    - 目的是赋予评审团能够帮助其他游戏更好更快地出现在最终排名环节的能力
    - 在具体的分值和最终排名的问题上,评审团没有其他权重
  - **评分人次示例**: 15个参赛者评分 + 3个评审团评分 = 15 + 2×3 = 21人次
  - **排名资格**: 总评分人次>20
  - **具体分值计算** (评审团与参赛者权重相等):
    ```
    某游戏的"创新性"维度:
    - 10个参赛者评分: [9, 8, 10, 7, 8, 9, 6, 10, 8, 7]
    - 2个评审团评分: [9, 10]

    最终平均分 = (9+8+10+7+8+9+6+10+8+7+9+10) ÷ 12
                = 所有评分的算术平均
                = 评审团的9分和参赛者的9分权重完全相等
    ```
  - **设计理由**:
    - 评审团成员是经过挑选的,评价更具权威性
    - 评审团的权重应该大于普通参赛者(在评分人次上)
    - ×2是简单明确的权重关系
  - **权重统一性**: 所有维度统一×2,不设差异化权重
  - **未评分处理**: 如果某评审团未给某游戏评分,按"×0"计算(即不计算该评审团的人次)
  - **未来考虑**: 可配置参数(如×1.5、×2、×3),但第一阶段固定×2
