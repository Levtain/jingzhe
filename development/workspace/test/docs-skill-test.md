# 文档Skill测试说明

> 这是一个测试文档,用于验证docs-write skill是否正常工作

## 测试目的

验证惊蛰计划项目中安装的3个文档处理Skill:
- `docs-write` - 文档写作
- `docs-review` - 文档审查
- `doc-coauthoring` - 协作文档编写

## docs-write 特点

根据skill定义,docs-write遵循以下写作原则:

### 1. 对话式风格

**✅ 应该这样写**:
- "你可以这样检查游戏评分..."
- "当你需要更新游戏时..."

**❌ 不要这样写**:
- "用户应该检查游戏评分..."
- "当用户需要更新游戏时..."

### 2. 读者友好

**匹配复杂度**:
- 不会过度简化复杂的概念
- 不会过度复杂化简单的概念

**快速给出答案**:
- 直接告诉用户怎么做
- 然后解释为什么

### 3. 清晰的结构

**标题应该具体**:
- ✅ "设置SAML后再添加用户"
- ❌ "SAML配置时机"

**先告诉做什么,再解释为什么**:
```
运行: `command-to-run`

然后: `next-command`

这确保你获取最新的更改。
```

## 使用示例

### 示例1: 编写技术文档

当你需要编写技术文档时,我会自动使用docs-write skill,确保:
- 语言口语化,像跟同事说话一样
- 重要信息放在前面
- 提供具体示例
- 代码示例可运行

### 示例2: 更新现有文档

更新文档时,我会:
- 保持现有风格一致
- 添加变更说明
- 检查交叉引用是否正确

### 示例3: 文档审查

使用docs-review skill时,我会检查:
- 语气是否过于正式
- 是否使用了"点击这里"这种链接
- 代码示例是否正确
- 格式是否一致

- ✅ 使用了对话式语言
- ✅ 结构清晰,有明确的章节
- ✅ 提供了具体的示例
- ✅ 使用了简洁的语言
