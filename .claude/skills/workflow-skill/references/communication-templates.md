# 沟通模板集合

本文档提供惊蛰计划项目中常用的沟通模板，可以直接使用或根据情况调整。

## 需求确认阶段模板

### 模板1：标准需求确认
```
【需求确认】
我理解你的需求是：
（用简单的话复述需求，1-2句话）

【使用场景】
这个功能会在这样的情况下使用：
（举一个具体的使用场景或例子）

【你的理解】
这就像...
（用生活化的类比）

【技术方案】
我打算这样实现：
1. 第一步：...
2. 第二步：...

【需要确认】
这样理解对吗？你同意这个方案吗？
```

### 模板2：需求不清晰时追问
```
【需求确认】
关于这个功能，我需要确认几个细节：

1. 【问题1】？
   - 选项A：...
   - 选项B：...
   你更倾向哪个？

2. 【问题2】？
   - 简单版：...
   - 完整版：...
   你觉得先做哪个版本？

【你的理解】
我理解的核心需求是：...
（用一句话总结）

这样理解对吗？
```

### 模板3：复杂需求的分解
```
【需求分析】
这个需求包含几个部分：

**第一部分**：...
（简单说明）

**第二部分**：...
（简单说明）

**第三部分**：...
（简单说明）

【实施建议】
我建议分步骤实现：
1. 第一步：先做第一部分（最核心）
2. 第二步：再做第二部分
3. 第三步：最后做第三部分

这样每次都有可用的功能，你觉得可以吗？
```

## 使用说明

### 如何使用这些模板

1. **选择合适的模板**
   - 根据沟通场景选择
   - 可以混合使用多个模板

2. **填充内容**
   - 括号里的内容是占位符
   - 根据实际情况填写

3. **调整语气**
   - 保持友好专业
   - 用简单直接的语言

4. **个性化**
   - 根据具体情况调整
   - 添加必要的细节

### 模板使用原则

- ✅ 灵活使用，不必死板
- ✅ 简洁明了，避免冗长
- ✅ 保持真诚，不要机械化
- ✅ 及时反馈，不要拖延
