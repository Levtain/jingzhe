## 设计概述

### 页面目标

评分页面是用户对游戏进行评分和打赏的核心交互页面。该页面需要:

1. ✅ 展示游戏基本信息（快速预览）
2. ✅ 提供6个维度的评分界面（滑块+悬浮提示）
3. ✅ 集成打赏功能（强制打赏机制）
4. ✅ 支持部分评分（维度可选填）
5. ✅ 提供评分修改功能（预填充已有评分）
6. ✅ 显示金币余额（实时更新）

### 核心设计原则

1. **简洁高效**: 减少操作步骤，提高评分效率
2. **即时反馈**: 滑块调整时实时显示分数和标准
3. **二次确认**: 防止误提交，提供确认弹窗
4. **引导友好**: 悬浮提示显示评分标准，无需额外培训
5. **成本透明**: 清晰显示打赏金额和金币余额

### 模块2: 评分区域 (核心)

#### 2.1 评分滑块设计

**每个维度的布局**:
```
┌────────────────────────────────────────────┐
│ 创新性                   [━━━━━━━━●━━━] 8.5分│
│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│ 是否新颖?                                  │
│ 游戏玩法、机制的原创性和突破性              │
│                                            │
│ 💡 鼠标悬浮滑块查看评分标准                 │
└────────────────────────────────────────────┘
```

**组件结构**:
1. **维度名称**: 左侧，加粗显示
2. **评分滑块**: 中间，可拖动
3. **当前分数**: 右侧，实时更新
4. **维度问题**: 副标题，提示该维度的核心关注点
5. **维度描述**: 维度简要说明（1-2行）
6. **悬浮提示**: 鼠标悬浮滑块时显示

#### 2.2 滑块交互规则

**默认值**:
- 首次评分: 默认5.0分（中间值）
- 修改评分: 预填充已有评分

**操作方式**:
- 点击滑块: 跳转到点击位置
- 拖动滑块: 实时更新分数显示
- 键盘操作: 左右箭头微调（0.5分步进）

**步进规则**:
- 支持0.5分档位
- 范围: 0-10分
- 最小值: 0分
- 最大值: 10分

**即时反馈**:
- 滑块拖动时 → 右侧分数实时更新
- 滑块拖动时 → 滑块颜色变化（根据分数档位）
- 滑块拖动时 → 底部悬浮提示实时更新内容

#### 2.3 悬浮提示设计

**触发时机**: 鼠标悬浮滑块

**展示内容** (根据当前分数动态变化):
```
┌─────────────────────────────────────┐
│  8-9分: 高度创新                      │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│  有显著的新颖性                       │
│  在玩法或机制上有明显创新             │
└─────────────────────────────────────┘
```

**分数档位与提示内容对照表**:

| 分数档位 | 创新性 | 主题诠释 | 视觉效果 | 音乐音频 | 整体性 | 整活儿 |
|---------|-------|---------|---------|---------|--------|--------|
| 10分 | 革命性突破 | 完美诠释 | 艺术级 | 大师级 | 完美无缺 | 神作 |
| 8-9分 | 高度创新 | 深度诠释 | 优秀美术 | 优秀音频 | 完成度高 | 非常好玩 |
| 6-7分 | 有新意 | 基本切题 | 美术合格 | 音频合格 | 基本完整 | 还算好玩 |
| 4-5分 | 中规中矩 | 略有偏差 | 美术粗糙 | 音频简单 | 完成度低 | 一般 |
| 0-3分 | 缺乏创新 | 严重跑题 | 美术差 | 音频差 | 明显未完成 | 不好玩 |

**详细评分标准**: 参见[评分维度详细指南_v1.0.md](../评分维度详细指南_v1.0.md)

#### 2.4 AI使用提示

**触发条件**: 游戏申报使用AI的维度

**展示方式**:
```
┌────────────────────────────────────────────┐
│ 视觉效果                 [━━━━━━━━●━━━] 8.8分│
│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│ ⚠️ 该游戏使用AI生成美术                     │
│    此维度评分将不参与排名                   │
│                                            │
│ 美术好不好看?                               │
│ 美术风格、画面表现力(含AI使用情况)          │
└────────────────────────────────────────────┘
```

**样式**:
- 使用黄色警告图标 ⚠️
- 文字颜色: 灰色（弱化显示）
- 位置: 维度描述上方

**目的**:
- 提醒用户该游戏使用了AI
- 告知用户该维度不参与排名
- 避免用户误认为评分无效

### 模块4: 操作按钮区

#### 4.1 按钮布局

**位置**: 页面底部，固定或浮动

**展示内容**:
```
┌────────────────────────────────────────────┐
│  [取消]  [暂存草稿]  [提交评分] ⭐          │
└────────────────────────────────────────────┘
```

#### 4.2 按钮功能

**[取消]**:
- 点击 → 弹出确认对话框
- 内容: "确定要取消评分吗？未提交的内容将丢失"
- 确认 → 跳转到游戏详情页

**[暂存草稿]**:
- 点击 → 保存当前评分到本地存储
- 不扣除金币
- 下次打开页面时恢复草稿
- 显示提示: "草稿已保存"

**[提交评分]** (主要按钮):
- 点击 → 触发二次确认弹窗
- 验证所有必填项
- 扣除金币
- 提交评分到服务器
- 成功后跳转到游戏详情页

#### 4.3 按钮状态

**禁用状态**:
- **[提交评分]禁用条件**:
  - 评分阶段未开始
  - 评分阶段已结束
  - 用户金币不足
  - 正在提交中（防止重复提交）

## 修改评分模式

### 场景: 用户已有评分，点击"修改评分"

#### 5.1 页面状态

**URL变化**: `/games/:id/score?type=edit`

**页面差异**:
- 标题: "修改评分" 而非 "开始评分"
- 预填充: 所有维度预填充已有评分
- 打赏金额: 预填充已有打赏金额
- 按钮: "[提交评分]" → "[更新评分]"

#### 5.2 预填充逻辑

**评分预填充**:
- 查询用户已有评分记录
- 填充所有维度的分数
- 未评分的维度保持默认值5.0分

**打赏预填充**:
- 查询用户已有打赏记录
- 填充打赏金额
- 显示提示: "修改评分不会额外扣除金币"

#### 5.3 修改评分的特殊提示

**打赏金额说明**:
```
┌────────────────────────────────────────────┐
│  💡 修改评分说明                            │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│                                             │
│  • 修改评分不会额外扣除金币                  │
│  • 打赏金额保持不变                         │
│  • 可随时修改，直到评分阶段结束              │
└────────────────────────────────────────────┘
```

**按钮文字**:
- "[提交评分]" → "[更新评分]"
- 颜色保持一致（主要按钮样式）

## 权限控制

### 评分权限矩阵

| 用户角色 | 可评分 | 可修改 | 限制 |
|---------|-------|--------|------|
| 路人(未登录) | ❌ | ❌ | 需登录 |
| 路人玩家 | ❌ | ❌ | 仅参赛者可评分 |
| 参赛者 | ✅ (非自评) | ✅ (非自评) | 不能评自己的游戏 |
| 评审团 | ✅ | ✅ | 可评所有游戏 |
| 管理员 | ❌ | ❌ | 管理权限，不参与评分 |

### 自评检测

**检测规则**:
1. 个人参赛: 用户ID == 游戏作者ID → 禁止评分
2. 团队参赛: 用户ID in 团队成员列表 → 禁止评分

## 数据验证

### 客户端验证

**提交前验证**:
1. ✅ 用户已登录
2. ✅ 用户有评分权限（参赛者/评审团）
3. ✅ 不是自评
4. ✅ 评分阶段进行中
5. ✅ 至少对1个维度进行评分
6. ✅ 金币余额 ≥ 打赏金额

**验证失败处理**:
- 显示错误提示
- 高亮错误字段
- 禁用提交按钮

### 服务器端验证

**提交时验证**:
1. ✅ 所有客户端验证项
2. ✅ 防止重复提交（幂等性检查）
3. ✅ 评分次数限制（每个游戏只能评1次）
4. ✅ 打赏金额合法性（1-10金币）
5. ✅ 金币余额充足（实时查询）

**验证失败处理**:
- 返回错误码和错误信息
- 回滚事务（如有）
- 记录错误日志

## 性能优化

### 页面加载优化

**首屏加载**:
1. 立即加载: 游戏基本信息、用户已有评分（修改模式）
2. 延迟加载: 评分标准悬浮提示内容
3. 按需加载: 金币余额（异步API）

**骨架屏**:
- 页面加载时显示骨架屏
- 数据加载完成后替换为实际内容

**本地存储**:
- 草稿保存到本地存储
- 下次打开页面时恢复草稿

### 交互优化

**防抖节流**:
- 滑块拖动事件: 节流（每100ms更新一次）
- 窗口resize事件: 防抖（延迟300ms）

**预加载**:
- 预加载评分标准提示内容
- 预加载金币余额

**缓存策略**:
- 游戏基本信息: 缓存5分钟
- 用户已有评分: 缓存1分钟（修改模式）

### 桌面端 (≥1024px)

- 评分滑块横向排列
- 打赏选项横向排列（单选按钮）

- 评分滑块保持横向排列
- 打赏选项改为2列布局
- 按钮区保持居中

### 移动端 (<768px)

- 评分滑块保持横向排列
- 打赏选项改为单列布局
- 按钮区改为全宽按钮
- 固定底部操作栏（提交按钮）
