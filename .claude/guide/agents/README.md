## 🤖 workflow-orchestrator-agent

### 何时使用

- 用户说："开始工作"、"接下来做什么"、"继续项目"
- 需要分析当前项目状态
- 需要推荐下一步行动
- 需要协调多个Agent

### 会做什么

1. **分析项目状态**
   - 读取 questions.md → 统计进度
   - 读取 claude.md → 了解当前阶段
   - 检查 blockers → 识别阻碍

2. **推荐下一步行动**
   - 根据优先级排序
   - 考虑依赖关系
   - 评估工作量

3. **自动启动Agent**
   - design-discussion-agent（讨论设计问题）
   - 其他专业Agent

### 使用方式

**自然语言**：
```
开始工作
接下来做什么
继续项目
```

**Agent会**：
- 自动分析状态
- 推荐任务
- 启动合适的Agent

## ⚡ 命令工具

### /discuss - 问题讨论

**何时使用**：
- 需要讨论设计问题
- 需要确认需求细节
- 需要做出决策

**会做什么**：
- 加载问题清单
- 逐个讨论问题
- 记录决策结果

**使用方式**：
```
/discuss                              # 默认推荐模块
/discuss 推荐位机制                   # 指定模块
/discuss 社区功能 --continue          # 继续上次讨论
```

---

### /sync-docs - 文档同步

**何时使用**：
- 问题讨论完成后
- 需要更新设计文档
- 需要同步到CHANGELOG

**会做什么**：
- 读取 questions.md
- 提取已确认问题
- 更新设计文档
- 更新 CHANGELOG.md

**使用方式**：
```
/sync-docs                            # 同步所有已确认问题
/sync-docs 推荐位机制                  # 只同步指定模块
/sync-docs --confirm                  # 同步前逐个确认
```

---

### /check-progress - 查看进度

**何时使用**：
- 想了解当前进度
- 想看已完成模块
- 想看待讨论问题

**会做什么**：
- 统计问题数量
- 计算完成百分比
- 列出已完成/待讨论模块

**使用方式**：
```
/check-progress                       # 显示完整进度
/check-progress --brief               # 简短摘要
/check-progress --module 排名系统     # 查看特定模块
```

---

### /check-doc-quality - 文档质量检查

**何时使用**：
- 文档修改后
- 发布前检查
- 定期质量审查

**会做什么**：
- 检查格式问题
- 检查版本号
- 检查引用链接
- 检查完整性

**使用方式**：
```
/check-doc-quality                    # 检查所有文档
/check-doc-quality docs/product/claude.md  # 检查单个文件
/check-doc-quality --fix              # 自动修复
```

---

### /task-planner - 任务规划

**何时使用**：
- 开始新阶段
- 需要制定计划
- 需要拆分任务

**会做什么**：
- 分析当前状态
- 制定任务计划
- 拆分任务步骤
- 估算工作量

**使用方式**：
```
/task-planner                         # 生成整体计划
/task-planner --week                  # 本周计划
/task-planner 推荐位机制              # 模块计划
```

---

### /check-completion - 完整性检查

**何时使用**：
- 阶段性检查
- 发布前验证
- 质量评估

**会做什么**：
- 检查模块完整性
- 检查问题覆盖率
- 识别遗漏内容

**使用方式**：
```
/check-completion                     # 检查所有模块
/check-completion 排名系统            # 检查特定模块
```

---

### /daily-summary - 每日总结

**何时使用**：
- 工作结束前
- 生成工作报告
- 计划下一步

**会做什么**：
- 记录完成的任务
- 统计进度增长
- 规划下一步行动

**使用方式**：
```
/daily-summary                        # 生成总结
/daily-summary --detailed             # 详细版
/daily-summary --save                 # 保存到文件
```

## 💡 使用技巧

### 技巧1：优先使用Agent

```
✅ 好的: "开始工作"
   → Agent自动分析、推荐、启动

❌ 不好: 手动查看文档、手动思考下一步
```

### 技巧2：用自然语言

```
✅ 好的: "看看进度到哪了"
❌ 不好: "/check-progress"
```

**AI会自动理解意图，调用相应工具**

### 技巧3：组合使用

```
你: 开始工作
AI: [调用workflow-orchestrator-agent]
    推荐讨论"推荐位机制"

你: 好
AI: [调用/discuss 推荐位机制]

[讨论完成后]

AI: 讨论完成！要同步文档吗？
你: 好
AI: [调用/sync-docs]
```

## 📊 工具对比

| 工具类型 | 优点 | 缺点 | 适用场景 |
|---------|------|------|---------|
| Agent | 智能分析、自动推荐 | 需要启动时间 | 开始工作、规划任务 |
| 命令 | 快速直接 | 需要手动指定 | 明确知道要做什么 |

**建议**：
- 🎯 不确定做什么 → 使用Agent
- ⚡ 明确要做什么 → 使用命令
