# 错误追踪与分析 Skill

## 概述

这个skill用于自动记录、分析工作中的错误，并形成改进措施。它帮助Claude持续学习和优化。

## 使用场景

在以下情况下应该使用error-tracker:

1. **用户指出错误时** - 当用户说"这里错了"、"格式有问题"等
2. **自己发现问题时** - 工作过程中发现的错误
3. **Code Review时** - 检查代码/文档时发现的问题
4. **工作流失败时** - 任务执行失败或结果不符合预期

## 工作流程

### 1. 记录错误

当发现错误时，立即记录到错误日志:

```markdown
#### [ERR-YYYYMMDD-NN] - 错误简短描述

**发现时间**: {{当前时间}}
**发现方式**: 用户反馈/自检/Code Review
**错误类型**: 格式问题/理解偏差/技术实现/沟通问题/流程问题
**严重程度**: 🔴严重/🟡中等/🟢轻微

**问题描述**:
- 详细的错误描述
- 用户的反馈内容
- 错误的具体表现

**根本原因分析**:
- 为什么会发生
- 哪里考虑不周
- 流程中的哪个环节出问题

**影响范围**:
- 影响了哪些文件/功能
- 对用户的影响程度

**解决方案**:
- 采取的修复措施
- 代码/文档变更

**预防措施**:
- 如何避免再次发生
- 需要添加的检查点
- 流程改进建议

**相关文件**:
- 涉及的文件路径

**状态**: ⏳待处理/🔄处理中/✅已解决
```

### 2. 更新统计

每次记录错误后，更新错误统计部分:

- 总错误数 +1
- 本周新增 +1
- 按类型分类统计 +1
- 按严重程度统计 +1

### 3. 分析模式

每周分析一次错误模式:

- 哪类错误最多?
- 哪些错误反复出现?
- 哪些预防措施有效?

### 4. 生成改进建议

基于错误分析，生成系统性改进建议:

- 需要开发的skill
- 需要添加的检查点
- 需要优化的流程
- 需要注意的事项

## 错误分类

### 格式问题
- markdown格式错误
- 代码风格不一致
- 文档结构问题
- 标点符号混用

### 理解偏差
- 误解用户需求
- 对要求理解不完整
- 缺少必要信息就开始工作

### 技术实现
- 代码bug
- 脚本执行失败
- 库/工具使用错误
- 依赖问题

### 沟通问题
- 没有确认需求
- 没有及时反馈进度
- 解释不清楚

### 流程问题
- 跳过必要步骤
- 顺序错误
- 缺少验证

## 严重程度

### 🔴 严重 (Critical)
- 影响最终交付
- 导致用户无法使用
- 需要立即修复

### 🟡 中等 (Medium)
- 不影响功能但影响体验
- 需要修复但不紧急

### 🟢 轻微 (Minor)
- 可以优化的地方
- 不影响使用

## 错误ID规则

格式: `ERR-YYYYMMDD-NN`

- `YYYY`: 年份
- `MM`: 月份
- `DD`: 日期
- `NN`: 当天第几个错误 (01-99)

示例: `ERR-20260111-01`

## 预防措施类型

### 检查点
- 在任务开始前添加检查
- 在任务完成后添加验证
- 在关键步骤添加确认

### 流程改进
- 优化工作流程
- 调整任务顺序
- 添加必要的步骤

### Skill开发
- 创建专门的skill处理常见问题
- 开发自动化检查工具
- 建立最佳实践指南

### 文档完善
- 更新文档说明
- 添加注意事项
- 记录常见错误

## 示例

### 记录错误示例

```markdown
#### [ERR-20260111-01] - markdown表格格式错误

**发现时间**: 2026-01-11 14:30
**发现方式**: 用户反馈
**错误类型**: 格式问题
**严重程度**: 🟡中等

**问题描述**:
- 用户指出"1.2共识2中的表格格式还是不对"
- 检查发现使用的是代码块包裹的文本表格，而不是标准markdown表格

**根本原因分析**:
- 没有仔细检查表格格式
- 使用了代码块```来包裹表格内容
- 没有使用markdown标准表格语法 |列1|列2|

**影响范围**:
- gamejam-planning-lecture-outline.md 第161-172行
- 转换到Word时表格格式不正确

**解决方案**:
- 将代码块改为标准markdown表格
- | 表头1 | 表头2 |
- |-------|-------|
- | 内容1 | 内容2 |

**预防措施**:
- ✅ 表格必须使用markdown标准语法
- ✅ 代码块仅用于真正的代码
- ✅ 在转换Word前检查表格格式
- ✅ 添加格式检查到工作流程

**相关文件**:
- d:\Claude\development\teaching\gamejam-planning-lecture-outline.md

**状态**: ✅已解决
```

## 每日使用

3. **工作结束后** - 更新统计，总结教训
4. **每周日** - 分析一周错误，生成改进计划
