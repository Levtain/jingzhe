## ✨ 什么是自然语言交互系统？

**核心变化**: 你不再需要记住命令了！

### 旧方式（v1.18及之前）

```bash
/check-progress    # 需要记住命令
/sync-docs         # 需要记住命令
/daily-summary     # 需要记住命令
```

❌ 需要记住21个命令
❌ 每次都要打"/"前缀
❌ 不符合自然交流习惯

### 新方式（v1.19）

```
看看进度到哪了      # 自然语言
同步一下            # 自然语言
今天先这样          # 自然语言
```

✅ 用自然语言即可
✅ 不需要记住命令
✅ 对话流畅自然

## 🎯 快速示例

### 查询进度

**你说**:
```
看看进度到哪了
```

**AI响应**:
```
好的，让我查看一下...

📊 当前进度：96/149 (64%)

今天完成了3个问题，进度不错！
```

### 完成任务

**你说**:
```
这个问题确认了
```

**AI响应**:
```
这个问题确认了！

要同步文档吗？
我会调用 /sync-docs 将已确认问题同步到设计文档
（大约需要1-2分钟）
```

### 结束会话

**你说**:
```
今天先这样
```

**AI响应**:
```
好的！今天完成了不少工作 🎉

要生成今日总结吗？
我会：
- 记录完成的任务
- 更新进度
- 规划下一步
```

## 🔄 工作原理

```
你: 看看进度到哪了
    ↓
AI: 意图识别 → "查询进度"
    ↓
AI: 映射工具 → /check-progress
    ↓
AI: 静默调用（无需确认）
    ↓
AI: 自然显示结果
```

**关键**:
- AI自动理解你的意图
- AI自动调用相应工具
- AI保持对话自然流畅
- 你不需要知道命令的存在

## 🎨 三种执行模式

### 1. 静默执行（查询类）

**特点**: 直接执行，无需确认

**示例**:
```
你: 看看进度
AI: [直接调用工具，显示结果]
```

**适用场景**: 查询信息，不修改文件

### 2. 确认执行（修改类）

**特点**: 先询问，再执行

**示例**:
```
你: 同步一下
AI: 要同步文档吗？
    我会调用 /sync-docs...
    （大约需要1-2分钟）
你: 好
AI: [调用工具]
```

**适用场景**: 修改文件，需要时间的操作

### 3. 智能推荐（建议类）

**特点**: 任务完成后主动推荐

**示例**:
```
你: 这个问题确认了
AI: 这个问题确认了！
    要同步文档吗？
```

**适用场景**: 任务完成、会话结束

---

## 🚫 什么时候不会推荐工具

AI不会在以下情况推荐工具：

- ❌ 你正在深入思考时
- ❌ 你刚明确拒绝过
- ❌ 你只是闲聊（"你好"、"谢谢"）
- ❌ 你只是在浏览/查看文档

## 💡 使用技巧

### 技巧1: 直接说你想做的

```
✅ 好的: "看看进度"
❌ 不好: "/check-progress"
```

### 技巧2: 保持对话自然

```
✅ 好的: "这个问题确认了"
❌ 不好: "请执行问题确认操作"
```

### 技巧3: 等AI推荐

```
你: 这个问题确认了
AI: 这个问题确认了！
    要同步文档吗？
你: 好        ← AI推荐的时机很合适
```

### 技巧4: 也可以拒绝

```
AI: 要同步文档吗？
你: 不需要     ← 完全可以拒绝
AI: 好的 👌    ← AI会理解
```

## 🐛 发现问题？

### 常见问题

**Q: AI还是说"我将调用XXX命令"**

A: SessionStart Hook没有正确加载。重启Claude Code试试。

**Q: AI没有推荐工具**

A: 可能不在推荐时机。等明确的完成信号（"好了"、"确认了"）。

**Q: AI过度推荐**

A: 记录具体场景，反馈以便优化。
