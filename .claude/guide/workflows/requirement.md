# 需求确认流程

> **如何高效地描述需求、确认方案**

---

## 🎯 流程概览

```
你描述需求 → 我确认理解 → 我给出方案 → 你确认 → 进入开发
```

---

## 📋 步骤1：你描述需求

### 推荐格式（不是强制）

```
【需求类型】新功能 / 修改 / 问题修复
【场景描述】在什么情况下...
【期望效果】我希望...
【优先级】高 / 中 / 低（可选）
```

### 示例1：新功能

```
【需求类型】新功能
【场景描述】参赛者提交游戏后
【期望效果】应该能看到自己的游戏在列表中，并且能看到有多少人评分
【优先级】高
```

### 示例2：体验优化

```
【需求类型】修改
【场景描述】用户浏览游戏列表时
【期望效果】希望能按评分高低排序，这样容易找到好游戏
【优先级】中
```

### 示例3：问题修复

```
【需求类型】问题修复
【场景描述】用户点击"提交评分"按钮后
【期望效果】应该显示提交成功
【实际效果】现在没有任何提示，不知道是否成功
【优先级】高
```

### 💡 不确定怎么描述？

直接说你想做什么即可：

```
我想要一个功能，让用户可以...
我觉得这个地方有点怪，但是不知道怎么说...
我发现有个bug，就是...
```

**我会追问细节**，直到我理解清楚。

---

## 📋 步骤2：我确认理解

### 我会这样做

1. **确认需求类型**（新功能/修改/问题）
2. **用产品经理能理解的方式复述确认**
3. **提供技术方案说明**（用"人话"解释）
4. **等待你确认**

### 我的回复格式

```
【需求确认】
我理解你的需求是：
[复述你的需求，用你能理解的方式]

【技术方案】
我打算这样做：
1. [方案要点1]
2. [方案要点2]
3. [方案要点3]

【类比说明】
这就像[生活中的类比]，让你更容易理解

【需要确认】
这样理解对吗？你同意这个方案吗？
```

### 示例

```
【需求确认】
我理解你的需求是：
参赛者提交游戏后，在游戏列表中能看到自己的游戏，
并且显示有多少人给这个游戏评过分。

【技术方案】
我打算这样做：
1. 在提交成功后，跳转到游戏列表页
2. 用一个特殊的标记突出显示自己的游戏（比如黄色边框）
3. 显示评分人数（例如：12人已评分）

【类比说明】
这就像你在电商网站买东西后，订单列表会显示你的订单状态，
让你一眼就能看到哪些是你买的。

【需要确认】
这样理解对吗？你同意这个方案吗？
```

---

## 📋 步骤3：你确认方案

### 情况A：完全同意

```
好，就这样做
```

→ 我立即开始开发

### 情况B：需要调整

```
整体可以，但是XX地方我想改一下...
```

→ 我调整方案，再次确认

### 情况C：有疑问

```
我不太明白XX部分，能再解释一下吗？
```

→ 我用更简单的方式解释

---

## 📋 步骤4：进入开发

**确认方案后**，我会：

1. ✅ 使用TodoWrite创建任务清单
2. ✅ 开始实施开发
3. ✅ 定期向你汇报进度
4. ✅ 遇到决策点立即询问

**你可以**：

- 📋 查看当前任务清单
- 📊 要求查看进度
- 🔄 提出修改意见
- ⏸️ 叫停某个功能
- ➕ 添加新需求

## 💡 沟通技巧

### 对产品经理（蜡烛先生）

**好的描述**:
- ✅ 用自然语言说你想做什么
- ✅ 描述具体场景（"用户点击XX后"）
- ✅ 说明期望效果（"我希望看到XX"）
- ✅ 不确定时就说"不确定"

**不需要**:
- ❌ 不需要懂技术术语
- ❌ 不需要考虑怎么实现
- ❌ 不需要写详细文档

### 对技术开发（Claude）

**好的理解**:
- ✅ 用产品经理能理解的方式复述
- ✅ 用类比说明（"这就像XX"）
- ✅ 提供完整方案，不要让产品经理选技术细节
- ✅ 不确定时追问

**避免**:
- ❌ 不要用技术术语
- ❌ 不要让产品经理做技术决策
- ❌ 不要假设理解，不确定就问

## 🎯 常见问题

### Q：我描述不清楚怎么办？

直接说：
```
我想要XX功能，但是不知道怎么描述清楚...
```

**我会**：
- 追问细节
- 提供选项
- 或给出建议方案

### Q：我不懂技术，怎么确认方案？

**不需要懂技术**！

我给出的方案会这样描述：
- "用户会看到..."（效果）
- "系统会自动..."（行为）
- "这就像XX"（类比）

你只需要确认：**这是你想要的效果吗？**

### Q：开发过程中可以改需求吗？

**可以**，随时告诉我：

```
我觉得XX地方需要改一下...
我想添加一个新功能...
```

**我会**：
- 评估修改的影响
- 告诉你需要多久
- 实施修改

---

## 📊 成功标准

### 好的需求确认

- ✅ 双方理解一致
- ✅ 方案清晰明确
- ✅ 你知道会得到什么效果
- ✅ 我知道要做什么

### 坏的需求确认

- ❌ 理解有偏差
- ❌ 方案模糊不清
- ❌ 期望不明确
- ❌ 开发后发现不是想要的
