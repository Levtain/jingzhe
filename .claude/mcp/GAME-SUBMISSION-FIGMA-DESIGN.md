## 📐 设计概述

### 页面结构

游戏提交页面采用**单页长表单 + 左侧导航**的设计，共7个步骤（含3个中间步骤）：

```
┌─────────────────────────────────────────────────────────┐
│  惊蛰 - 游戏提交                          [保存草稿][预览][提交] │
├────────────┬────────────────────────────────────────────┤
│            │  步骤1: 游戏标题                             │
│  左侧导航   │  ┌──────────────────────────────────────┐  │
│            │  │ 游戏标题                              │  │
│  1. 标题 ✓  │  │ [_____________________________]       │  │
│  2. 简介    │  │                                      │  │
│  3. 正文    │  │ 5-100个字符 | 当前: 25/100 字符      │  │
│  4. 媒体    │  └──────────────────────────────────────┘  │
│  5. 链接    │                                             │
│  6. 维度    │  步骤2: 游戏简介                             │
│  7. 确认    │  ┌──────────────────────────────────────┐  │
│            │  │ 游戏简介（最多100字符，可选）         │  │
│            │  │ [_____________________________]       │  │
│            │  │                                      │  │
│            │  │ 当前: 0/100 字符                      │  │
│            │  └──────────────────────────────────────┘  │
│            │                                             │
│            │  ... (更多表单内容)                         │
└────────────┴─────────────────────────────────────────────┘
```

## 🎨 设计规范

### 颜色方案

**主色调**:

### 创建步骤

**1. 创建页面结构**
- 创建Frame: 1440 × 900 (桌面端)

**2. 创建组件库**
- Button - Primary/Secondary/Danger
- Input - Default/Focus/Error
- Checkbox - Default/Checked/Disabled
- File Upload - Empty/Filled
- Navigation Item - Active/Inactive

**3. 创建表单区块**
- 每个步骤创建一个Component
- 使用Variants处理不同状态
- 添加交互原型

**4. 添加交互原型**
- 点击导航项 → 滚动到对应区域

## 💡 设计亮点

1. **单页长表单** - 用户可预览全局，流畅填写
2. **左侧导航** - 快速跳转，进度清晰
3. **实时验证** - 即时反馈，减少提交后错误
4. **自动保存** - 防止数据丢失
5. **草稿预览** - 提前查看最终效果
6. **响应式设计** - 桌面端和移动端体验优化
