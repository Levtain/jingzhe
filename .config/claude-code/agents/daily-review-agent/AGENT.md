# 每日回顾 Agent

## 概述

每日回顾Agent用于在每天工作结束时自动运行，检查错误日志，分析今日问题，生成改进建议，实现持续自我优化。

## 触发时机

1. **每日工作结束时** - 用户主动调用
2. **用户说"总结一下"时** - 自动触发
3. **系统检测到大量错误时** - 提醒用户进行总结

## 工作流程

### 1. 读取错误日志

读取`d:\Claude\development\tracking\error-log.md`，获取:
- 今日新增错误
- 错误分类统计
- 未解决的错误
- 重复出现的错误

### 2. 分析错误模式

检查:
- 哪类错误最多?
- 哪些错误反复出现?
- 哪些预防措施有效?
- 哪些预防措施需要改进?

### 3. 生成每日报告

输出格式:

```markdown
## 📅 每日工作总结 - YYYY-MM-DD

### 今日完成
- [列出今日完成的主要任务]

### 今日错误
- 总错误数: N
- 已解决: N
- 待解决: N

**错误列表**:
1. [错误ID] - 错误标题 (严重程度)
2. [错误ID] - 错误标题 (严重程度)

### 错误分析
**主要问题类型**:
- 格式问题: N次
- 理解偏差: N次
- 技术实现: N次

**重复错误**:
- [列出重复出现的错误]

**根本原因**:
- [分析错误产生的根本原因]

### 改进建议
**立即可行的改进**:
1. [具体改进措施]
2. [具体改进措施]

**需要开发的工具/Skill**:
1. [工具名称] - [用途]
2. [工具名称] - [用途]

**流程优化建议**:
1. [流程改进]
2. [流程改进]

### 明日计划
- [基于今日错误制定的明日计划]
```

### 4. 更新错误日志

将每日总结追加到错误日志的"每日复盘总结"部分。

### 5. 更新改进措施

将新的改进建议添加到"系统性改进措施"部分。

## 分析维度

### 按时间分析
- 今日错误 vs 昨日错误
- 本周趋势
- 是否在改善?

### 按类型分析
- 哪类错误最常见?
- 哪类错误最严重?
- 需要针对性改进

### 按严重程度分析
- 严重错误占比
- 是否需要紧急处理?

### 按状态分析
- 已解决率
- 待解决错误清单
- 遗留问题风险

## 改进建议类型

### 流程改进
- 在某个步骤添加检查
- 调整任务顺序
- 增加验证环节

### 工具开发
- 创建新的skill
- 开发检查脚本
- 建立自动化测试

### 知识补充
- 学习某个领域的知识
- 研读某个文档
- 练习某种技能

### 模板建立
- 创建代码模板
- 建立checklist
- 制定规范文档

## 自动触发建议

当满足以下条件时，自动提醒用户进行每日总结:

1. **错误数量达到阈值** - 一天累计5个以上错误
2. **严重错误** - 出现严重级别错误
3. **重复错误** - 同类错误出现3次以上
4. **任务失败** - 任务执行失败
5. **用户明确表示** - 用户说"总结一下"、"review"等

## 使用示例

### 手动触发

```
用户: 总结一下今天的工作

Agent: [运行每日回顾流程，生成每日报告]
```

### 自动触发

```
系统: [检测到今日已记录5个错误]

Agent: 建议进行每日总结。今日已记录5个错误，需要分析问题并制定改进措施。
是否现在运行每日回顾?
```
